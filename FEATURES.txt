----------------------------------------------------------------
  18. AI FEATURES (Google Gemini)
----------------------------------------------------------------

  Service: services/ai_review.py
    - Collects IMDb user reviews via Serper (3 parallel queries)
    - Sends reviews to Gemini for structured analysis
    - Returns: summary, sentiment, rating estimate, pros, cons,
      highlights, audience verdict, recommended_for
    - Skips unreleased / upcoming movies automatically
    - Fallback: builds summary directly from Serper snippets
      (no hallucination — only what reviews say)
    - 24-hour cache
    - Tries 4 Gemini models in order (quota fallback)

  Service: services/ai_recommend.py
    - Fetches TMDB keywords, similar movies, recommendations
    - Collects web context via Serper (audience opinions)
    - Sends all data to Gemini for ranked recommendations
    - Returns: reason, recommendations with match_score (0-100)
    - Fallback: TMDB-based recommendations if Gemini fails
    - Enriches results with poster/backdrop URLs
    - 24-hour cache

  Service: services/ai_search.py
    - Extracts structured filters from natural language query
    - Filters: language, genre, year, actor, director, OTT,
      sort, runtime, rating, similar_to, free_text
    - Builds TMDB Discover query from filters
    - Person search (actor/director name → TMDB person ID)
    - Similar movie search
    - Serper OTT enrichment
    - Attaches AI reason to each result
    - Telugu-only filter enforced
    - 30-minute cache
    - Fallback: rule-based NLP parser

  Service: services/ai_watchlist.py
    - Analyzes watchlist to build taste profile (Gemini)
    - Returns: taste_summary, favorite_genres, favorite_actors,
      favorite_directors, preferred_runtime, preferred_years, mood
    - Generates 5-category personalized recommendations (Gemini)
    - Enriches recommendations with TMDB poster/backdrop
    - Fallback: rule-based taste analysis + TMDB discover
    - Persists taste profile to SQLite user_preferences table

----------------------------------------------------------------
  23. PERFORMANCE & RELIABILITY
----------------------------------------------------------------

  - TMDB responses cached in-memory (5 min default)
  - AI responses cached 24 hours
  - YouTube responses cached 10 minutes
  - Serper responses cached 10 minutes
  - Smart search cached 10 minutes
  - Recommendation results cached 10 minutes
  - Person data cached 1 hour
  - Each API section loads independently (isolated try/except)
  - One section failure never blocks other sections
  - Retry logic on TMDB (5 retries, exponential backoff)
  - DNS + TCP reachability check before TMDB calls
  - Graceful degradation: AI fails → TMDB fallback
  - Gemini model fallback chain (4 models tried in order)
  - Lazy image loading (IntersectionObserver, 200px margin)
  - Skeleton loading shown immediately while data loads
  - Image fallback chain: poster → backdrop → local SVG
  - Parallel data fetching (ThreadPoolExecutor) for movie core
  - TMDB image proxy route for server-side image serving
